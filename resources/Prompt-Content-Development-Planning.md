# Brief for content development planning chat

I'm building a 12-week self-directed semantic web curriculum ("Sensemaking Semantic Web") published openly under my consulting brand Sensemaking AI. The curriculum has four modules:

1. Foundations (RDF, Turtle, basic SPARQL)  
2. Modeling (RDFS, OWL, ontology design)  
3. Reasoning at the edge (inference, reification, SHACL)  
4. Shipping (SPARQL UPDATE, deployment, LLM \+ KG)

The full syllabus, module READMEs, and homebase HTML are all built. What's missing is actual teaching content — explanations, walkthroughs, worked examples in my voice. The curriculum currently points at canonical references (Allemang, DuCharme, Hogan, W3C specs) but contributes no original teaching material.

I want to develop 2-3 sub-modules under each module, each rendered as an HTML page (8-12 pages total). Each page should be a coherent teaching unit that:

- Synthesizes the canonical material rather than duplicating it  
- Includes worked examples drawn from my projects (Resume Graph Explorer, Naruto Network Graph, the Digital Twin / TwinKit, the memorial graph)  
- Surfaces the pain perspective on the technology honestly  
- Curates external resources (videos, tutorials, papers) where they exist and are high-quality  
- Uses inline diagrams, code samples, and interactive widgets where they teach something  
- Connects to my broader practice (cog sci \+ applied AI \+ knowledge representation)

## What I want from this chat

A planning document — NOT actual content production. Specifically:

1. **A proposed sub-module breakdown** for all four modules. For each module, 2-3 sub-modules with:  
     
   - A clear topic and scope  
   - The 1-2 canonical references that ground it  
   - A concrete worked example drawn from my existing projects  
   - The specific "pain point" angle the sub-module surfaces

   

2. **A targeted web search** for each proposed sub-module, identifying:  
     
   - High-quality existing resources I should link to or build on  
   - Notable gaps where my content would genuinely add something new  
   - Any interactive tutorials or visualizations worth embedding

   

3. **For 2-3 sub-modules that seem highest-impact**, a more detailed page outline:  
     
   - Section structure  
   - Specific diagrams to include  
   - Specific code samples to walk through  
   - Whether an interactive widget would help, and what it would do  
   - Rough word count

   

4. **A rollout plan** — which sub-modules to build first, which can wait, and a realistic pace given that I'm also working through the curriculum's actual content week by week.

## What I don't want from this chat

- Don't produce the actual HTML for any sub-module yet  
- Don't write extensive prose drafts  
- Don't try to plan all 8-12 sub-modules at the same level of detail  
- Don't ignore the canonical references — the content should complement them, not replace them

## Voice and conventions

The curriculum has an established voice: declarative, contrastive, honest about complexity, cog-sci-flavored where it earns its place, no italics for emphasis (the voice does the work), no corporate- speak. The pages should match that voice.
